Bootstrap-sass on Rails

Components

Bootstrap comes with a ton of sensible defaults for typography.

Headings with secondary text.

Body copy and inline text elements.

Alignment classes, transformation classes, abbrevations, addresses, blockquotes and lists.

With the table classes you can really whip together a basic data displaying site that looks good.

All the official documentation for this stuff is great, you should read it, and there's no point in trying to recreating it here.

The rest of this chapter is going to discuss forms (since they can get a little tricky) and the bootstrap JavaScript components. The bootstrap.js has a number of jQuery plugins included which let you add rich functionality to your site. There are many other great jQuery plugins out there, but the ones included in bootstrap all work in similar ways and don't require you to write any javascript code to use them. (Though you can use them via javascript if you need to.)

These components all work in similar ways. To configure them you put additional data attributes on the element, and when the page is loaded the bootstrap.js will look for that and attach the various event listeners needed to make them work.

For example, a drop down menu could be implemented:

<div class="dropdown">
  <a class="downdown-toggle" id="drop" role="button" data-toggle="dropdown" href="#">Dropdown<span class="caret"></span></a>
  <ul id="menu2" class="dropdown-menu" role="menu" aria-labelledby="drop">
    <li role="presentation"><a role="menuitem" tabindex="-1" href="#">Action</a></li>
    <li role="presentation"><a role="menuitem" tabindex="-1" href="#">Another action</a></li>
    <li role="presentation"><a role="menuitem" tabindex="-1" href="#">Something else here</a></li>
    <li role="presentation" class="divider"></li>
    <li role="presentation"><a role="menuitem" tabindex="-1" href="#">Separated link</a></li>
  </ul>
</div>

The classes of dropdown, dropdown-menu, dropdown-toggle will define the overall structure of the elements effected. The attribute of data-toggle="dropdown" on the other hand is what enables the javascript code to be attached to that element.

Bootstrap Events

Bootstrap provides custom events for most plugins' unique actions. Generally, these come in an infinitive and past participle form - where the infinitive (ex. show) is triggered at the start of an event, and its past participle form (ex. shown) is triggered on the completion of an action.

As of 3.0.0, all Bootstrap events are namespaced.

All infinitive events provide preventDefault functionality. This provides the ability to stop the execution of an action before it starts.

For a dropdown the events could be something like:

  • show.bs.dropdown
  • shown.bs.dropdown
  • hide.bs.dropdown
  • hidden.bs.dropdown

Or for a modal dialog:

  • show.bs.modal
  • shown.bs.modal
  • hide.bs.modal
  • hidden.bs.modal

These events are based upon the jQuery event system. There are situations where you want to trigger another action based upon what the UI is doing, and you can do this by setting handlers on these events. These handlers look like:

$("#myModal").on('hide.bs.modal', function (e) {
  console.log( "We are hiding ourself!" );
} )

In the infinitive events, you can stop the action from happening, by calling e.preventDefault().

Bootstrap Event Logger (Esoteric)

If you are curious of all the jQuery events that are being triggered, you can add the following code to a development version of your site. This overrides the default jQuery method to first log the event to the console.

$( function() {
    $.fn.originalTrigger = $.fn.trigger;
    $.fn.trigger = function(type, data) {
      console.log(typeof(type)=='string' ? type : type.type, data);
      return $(this).originalTrigger.apply(this,arguments);
    }
} );

With this code in place, you won't see all of the browser events, but you will see all of the events that jQuery is sending around.
