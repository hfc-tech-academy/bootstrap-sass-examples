Bootstrap-sass on Rails

Modal

Modal dialog boxes are content that comes up above your main page. They look like this:

Lets look at some basic markup:

<!-- Button trigger modal -->
<button class="btn btn-primary btn-lg" data-toggle="modal" data-target="#myModal">
  Launch demo modal
</button>

<!-- Modal -->
<div class="modal fade" id="myModal" tabindex="-1" role="dialog" aria-labelledby="myModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
        <button type="button" class="close" data-dismiss="modal" aria-hidden="true">&times;</button>
        <h4 class="modal-title" id="myModalLabel">Modal title</h4>
      </div>
      <div class="modal-body">
        <h1>Hello World!</h1>
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-default" data-dismiss="modal">Close</button>
        <button type="button" class="btn btn-primary">Save changes</button>
      </div>
    </div><!-- /.modal-content -->
  </div><!-- /.modal-dialog -->
</div><!-- /.modal -->

Modals are generally not shown to the user by default. It's shown normally as a result of some action and you want to prompt them for more information before going forward. The part that causes the modal to display is called the "trigger". In the above code, we are triggering the modal using bootstrap's data-api, specifically the data-toggle="modal" which tells bootstrap that we want to turn on a modal dialog, and data-target="#myModal" which is the DOM id of our modal.

Lets look at the modal definition itself. If we strip everything down to the base class heirachy, it looks like;

.modal
  .modal-dialog
    .modal-content
      .modal-header
        .modal-title
      .modal-body
      .modal-footer

These are the basic classes that you need to have. The outer modal class is what gets positioned on the screen, as it is by default not shown. In our example, there are a number of other attributes listed.

<div class="modal fade" id="myModal" tabindex="-1" role="dialog" aria-labelledby="myModalLabel" aria-hidden="true">

fade means that, when the modal dialog is triggered, there's an opacity transition animation where it appears. id="myModal" is the idenfitication. tabindex="-1" is to remove the element from the default navigation flow. role="dialog", aria-hidden="true" and aria-labelledby="myModalLabel" are for screen readers, where we identify what the modal is about, and specify where the text to read to the user is located. (In this case, the modal-title has an id of myModalLabel.)

<div class="modal-dialog">

This is the container for the visible part of the modal. If you want a smaller one you can use class="modal-dialog modal-sm"

And a larger one is class="modal-dialog modal-lg".

The content of the modal needs to be wrapped in modal-content.

From there, we have the header, body, and footer. (Very similar to the ways that panels are defined.) Lets look at the header:

<div class="modal-header">
  <button type="button" class="close" data-dismiss="modal" aria-hidden="true">&times;</button>
  <h4 class="modal-title" id="myModalLabel">Modal title</h4>
</div>

There are a few attributes on the button element to make it work correctly. The class="close" is the styling/positioning, and data-dismiss="modal" is again part of the bootstrap data-api. This will dismiss the dialog. There's no data-target needed here because the button is inside of a modal. aria-hidden we've seen before, and this is again to optimize the behavior for screen readers, since &times; isn't meaningful.

The modal-title is the header text. The id specified here is the one that was referenced by the area-labelledby above.

Options

data-backdrop="static" this will disable clicking-off-to-close.

data-keyboard="false" this will disable pressing escape to close.

data-show="true" this will show the modal by default.

Triggering and loading remote content

If you want the content of the modal to be dynamic, one easy way to do this is to use a remote trigger. This is simply done by passing in an href or data-remote into the trigger:

<a class="btn btn-primary btn-lg" data-toggle="modal" data-target="#myModal" data-remote="url.html">My button!</a>

Triggering using JavaScript

The data-api is a simple way to trigger showing the modal dialog, but sometimes you want to show the dialog as result of a calculation. The basic way that this works is:

// To show a modal
$("#myModal").modal();

// Another way to show a modal
$("#myModal").modal('show');

// To toggle a modal
$("#myModal").modal('toggle');

// To hide a modal programmatically:
$("#myModal").modal('hide');

You can also configure the modal to behave different ways. By default, the user clicks off of the modal or presses the esc key and the modal will be dismissed. To change this, you pass in options to the modal method:

var options = {
  background: "static", // Disables clicking away to close
  keyboard: false, // Disables the escape key
  show: true, // This shows the modal by default
  remote: url // This loads a url
};

$("#myModal").modal(options);

Modal events

Bootstrap fires off events when the user interacts with the modal.

Event Name Description
show.bs.modal This event fires immediately when the show instance method is called. If caused by a click, the clicked element is available as the relatedTarget property of the event.
shown.bs.modal This event is fired when the modal has been made visible to the user (will wait for CSS transitions to complete). If caused by a click, the clicked element is available as the relatedTarget property of the event.
hide.bs.modal This event is fired immediately when the hide instance method has been called.
hidden.bs.modal This event is fired when the modal has finished being hidden from the user (will wait for CSS transitions to complete).
loaded.bs.modal This event is fired when the modal has loaded content using the remote option.

Lets take a look at how to use this:

$('#myModal').on('show.bs.modal', function (e) {
  console.log( "We've started to show our modal!" );
})
$('#myModal').on('shown.bs.modal', function (e) {
  console.log( "We've finished showing our modal!" );
})

$("#myModal").on('hide.bs.modal', function (e) {
  console.log( "We are hiding ourself!" );
} )

$('#myModal').on('hidden.bs.modal', function (e) {
  console.log( "User hid our modal!" );
})

Styling a modal to slide up from the bottom on a mobile device

Building a mobile site really improves the experience of your mobile users. It's not as good as a native application, but it can get very close. Responsive design helps with that, so lets see what we can do to make our modal dialogs look better on small devices.

Modals are generally used to prompt the user to make a significant action. They can cancel, but you want to pull the focus from what's going on to give them a few options. On iOS, this is normally done by sliding up a set of full width buttons from the bottom of the screen and graying out the content above.

We only want this to show on small screens, so we can start by repositioning the modal-dialog on small screens only:

@media ( max-width: $screen-xs-max ) {
  .modal-dialog {
    position: absolute;
    left: 0;
    right: 0;
    bottom: 0;
    margin: 0;
  }
}

Lets load this up, and make your viewpoint larger and smaller. You can see that when the modal is open and the window gets smaller, the modal is now at the bottom of the screen.

Lets now change the transition, so that instead of coming down from the top we will bring it up from the bottom:

@media ( max-width: $screen-xs-max ) {
  .modal-dialog {
    position: absolute;
    left: 0;
    right: 0;
    bottom: 0;
    margin: 0;
  }

  .modal {
    &.fade .modal-dialog {
      @include translate3d(0,100%,0);
    }
    &.in .modal-dialog { @include translate3d(0, 0, 0) }
  }
}

This uses some bootstrap mixins and CSS3 animations. When the fade class is set on the modal, it's shifted 100% down, and when the in class is added it transitions back to it's happy place.

The last thing we need to do is get rid of the rounded corners and borders to make the dialog flush with the sides. These styles are on modal-dialog and we're going to zero them out, giving us the final CSS:

@media ( max-width: $screen-xs-max ) {
  .modal-dialog {
    position: absolute;
    left: 0;
    right: 0;
    bottom: 0;
    margin: 0;
  }

  .modal-content {
    -webkit-border-radius: 0;
    border-radius: 0;
    border: none;
  }
  .modal {
    &.fade .modal-dialog {
      @include translate3d(0,100%,0);
    }
    &.in .modal-dialog { @include translate3d(0, 0, 0) }
  }
}
