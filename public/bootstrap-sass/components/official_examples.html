The first thing we see is

<nav class="navbar navbar-default" role="navigation">
  <div class="container-fluid">

Which defines a navbar and then a full width container inside. This will have the navbar go across the top of the page with no margins. Change this to container if you which you have it match up with the content area.

The next is the navbar-header:

    <div class="navbar-header">
      <button type="button" class="navbar-toggle collapsed" data-toggle="collapse" data-target="#bs-example-navbar-collapse-1">
        <span class="sr-only">Toggle navigation</span>
        <span class="icon-bar"></span>
        <span class="icon-bar"></span>
        <span class="icon-bar"></span>
      </button>
      <a class="navbar-brand" href="#">Brand</a>
    </div>

Here there is a button with the classes navbar-toggle and collapsed. navbar-toggle tells bootstrap that this element is pulled to the right and there is a media query rule to hide this on larger screens. The collapsed class here is actually unecessary and misleading since this is hidden or shown based upon the .navbar-toggle class.

There is also a data-toggle="collapse" on this with a data-target pointing to a different element, which will toggle displaying main navigation links. Remember that this button is only available if the screen is narrow.

    <div class="collapse navbar-collapse" id="bs-example-navbar-collapse-1">

When the screen is wide enough, there is a CSS rule to always display .collapse.navbar-collapse. This means that the collapse behavior is overridden from what you might expect for larger displayes. When the screen is narrow (at the same point that navbar-toggle is displayed, then the regular rules of the collapse component apply.

So, when the navbar is wide, the different navbar items are displayed as inline lists no matter what. When the navbar is narrow, then the container for the navbar items is collapsed by default. When the navbar-toggle component is triggered, it opens up the navbar and the menu is displayed. At this point, the navbar components are defined as blocks rather than inline, so they become vertically placed.
