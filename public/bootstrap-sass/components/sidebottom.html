Sidebottom

Lets build a responsive navigation bar that has large icons on the left when the window is large (on desktop or tablet):

and then has the navigation bar on the bottom when viewed on small devices.

This is the mark up we are going to use:

<div class="container-fluid">
  <div class="row">
    <div class="sidebottom">
      <ul class="nav">
        <li><a href="#"><span class="glyphicon glyphicon-home"></span><br/>Home</a></li>
        <li><a href="#"><span class="glyphicon glyphicon-user"></span><br/>Profile</a></li>
        <li><a href="#"><span class="glyphicon glyphicon-flag"></span><br/>Flagged</a></li>
        <li><a href="#"><span class="glyphicon glyphicon-star"></span><br/>Starred</a></li>
      </ul>
    </div>

    <div class="content">
       ...
    </div>
  </div>
</div>

The whole layout is wrapped in container-fluid and a row, and then we have two classes that we are going to put our content in: sidebottom and content. On larger screens, we are going to have sidebottom be 2 columns across and the content be 10 columns. We'll also adjust the font sizes for the nav icon and text:


.content {
  @extend .col-sm-10;
}

.sidebottom {
  @extend .col-sm-2;

  li {
    text-align: center;
    a {
      font-size: 18px;
      span {
        font-size: 45px;
      }
    }
  }

Here we're using the semantic CSS to define our classes, and making the content be col-sm-10 and the sidebottom be a col-sm-2. I've also adjusted the styling on the nav items themselves, to center the text, and up the sizes of the icons and the text.

The next step is to change the style of the navbar when the screen is smaller. We're going to use a media query to do this. The first thing we are going to do is to make the navbar horizontal, and adjust the sizing.

.sidebottom {
  @extend .col-sm-2;

  li {
    text-align: center;
    a {
      font-size: 18px;
      span {
        font-size: 45px;
      }
    }
  }

  @media (max-width: $screen-sm-min) {
    li {
      display: inline-block;
      text-align: center;
      width: 24%;
      a {
        font-size: 12px;
        span {
          font-size: 22px;
        }
      }
    }
  }
}

The @media (max-width: $screen-sm-min) line is where you set the breakpoint of when the rules apply. In this case, the maximum width of the screen needs to be $screen-sm-min, so the following rules only apply when the window is small.

We've picked 24% here as the width of each of the items since our sample HTML has 4 elements, and we round down slightly from 25% to make sure that everything fits.

At this point, the col-sm-2 has been displayed as a full width row, and the nav icons are above the content area. Let's change it. Due to the way that the navbar-fixed-bottom is implemented, we can't use an extend to bring in the properties, but they aren't that complicated:

.sidebottom {
  @extend .col-sm-2;

  li {
    text-align: center;
    a {
      font-size: 18px;
      span {
        font-size: 45px;
      }
    }
  }

  @media (max-width: $screen-sm-min) {
    background: white;
    position: fixed;
    bottom: 0;
    left: 0;
    right: 0;
    z-index: 100;

    li {
      display: inline-block;
      text-align: center;
      width: 24%;
      a {
        font-size: 12px;
        span {
          font-size: 22px;
        }
      }
    }
  }
}

Now the navbar is in the correct place, but it's covering the content of the body. We need to adjust the padding-bottom of the main body, but only when the window is small:

@media (max-width: $screen-sm-min) {
  body {
    padding-bottom: 26px;
  }
}

If we wanted to have a fixed header on the top, but only on small screens, we could add this HTML to the top of the file:

<nav class="navbar navbar-fixed-top visible-xs">
  <div class="container-fluid">
    <div class="row">
      <h1>It's the roof!</h1>
    </div>
  </div>
</nav>

The visible-xs class means that this element is only visible when the screen is small. These are the helper classes built into bootstrap so you don't need to write manual media queries that we did above. We also need to adjust the CSS a bit, to shift down the top of the page (so you can see the content underneath the topnav) and to fix the styling a bit.


@media (max-width: $screen-sm-min) {
  body {
    padding-top: 50px;
    padding-bottom: 26px;
  }

  nav {
    background: white;
    h1 {
      padding-top: 0px;
      margin-top: 5px;
    }
  }
}
