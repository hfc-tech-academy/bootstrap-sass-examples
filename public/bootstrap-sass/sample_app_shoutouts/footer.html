Footer

Lets start editing. First lets break out the page and start laying things out. We'll start with a basic app/views/layouts/application.html.erb. (If you are using seed, this will be in haml and will use partials).

<!DOCTYPE html>
<html class="no-js">
  <head>
    <meta charset="utf-8"/>
    <meta content="IE=edge,chrome=1" http-equiv="X-UA-Compatible"/>
    <%= display_meta_tags :site => 'Seed Site', :title => page_title, :reverse => true %>
    <meta name="description"/>
    <meta content="width=device-width" name="viewport"/>
    <%= stylesheet_link_tag 'application', :media => nil %>
    <% # Append your own using content_for :stylesheets %>
    <%= yield :stylesheets %>
    <%= javascript_include_tag 'modernizr' %>
    <%= csrf_meta_tag %>
  </head>
  <body>
    <!--[if lt IE 7 ]
      <p class="chromeframe">
        You are using an <strong>outdated</strong> browser.
        Please <a href="http://browsehappy.com/">upgrade your browser</a> or <a href="http://www.google.com/chromeframe/?redirect=true">activate Google Chrome Frame</a> to improve your experience.
      </p>
    -->

    <div class="container">
      <%= yield %>
      <% # Javascript at the bottom for fast page loading %>
      <%= javascript_include_tag 'application' %>
      <% #  Append your own using content_for :javascripts %>
      <%= yield :javascripts %>
    </div>
  </body>
</html>

Background Image

We're starting with the broad strokes of the design and working our way in, so put the background in place. That'll help make it look like a real site.

Assets for this project can be found at https://github.com/hfc-tech-academy/shoutouts/tree/master/app/assets

Edit app/assets/stylesheets/application.css.scss to be:

//-----------------------------------------
// Variables, customize bootstrap here
//-----------------------------------------

@import 'bootstrap-sprockets';
@import 'bootstrap';

body {
  background: image-url( "primary_bkg.jpg" ) #0092ce;
  background-repeat: repeat-x;

}

And reload the page. You should now see the page of the background. There is a lot going on here, here's what is listed:

@import 'bootstrap-sprockets';
@import 'bootstrap';

These two lines load up bootstrap. The @import command is a SASS command, which loads up the SASS bootstrap files that are included in the bootstrap-sass gem. We will be customizing bootstrap using variables, and these variables should be set before we import bootstrap itself. When the SASS is compiled, the variables that you set are used to control the way that the final CSS is generated. Therefore they need to be set before you include the bootstrap SASS source files.

body {
  background: image-url( "primary_bkg.jpg" ) #0092ce;
  background-repeat: no-repeat;
}

After bootstrap is included, we want to override some of the default. Specifically, we want to set the background image on the body tag. We set this using image-url. This is part of the Rails asset pipeline. In the rendered CSS, the url tag will reference the compiled version of the image url. If you don't use image-url and use url instead, you will probably have problems when running in production mode.

Footer

Each of the pages has the same footer. For this we are going to use the <footer> element, which is just like a div. Lets create a footer and put the list of links in there.

Here's the footer HTML (added below the container div)

    <footer>
      <ul>
        <li><a href="#">Help</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Sign in</a></li>
      </ul>
    </footer>

If we load the page at this point, you'll see the unstyled footer there. We want to make the links horizontal, and the text uppercase and bold, so lets add some css inside of app/assets/stylesheets/application.css.scss

footer {
  ul {
    list-style: none;
    padding-left: 0;
    li {
      display: inline;
      margin-right: 1em;
      a {
        text-transform: uppercase;
        font-weight: bold;
      }
    }
  }
}

Nested css rules is something that SASS gives you, which makes it easier to target the elements that you want without having to repeat yourself as much. This SASS translates into:

footer ul {
  list-style: none;
  padding-left: 0;
}

footer ul li {
  display: inline;
  margin-right: 1em;
}

footer ul li a {
  text-transform: uppercase;
  font-weight: bold;
}

Keeping the rules nested keeps things cleaner overtime, and means that you repeat yourself less. Which is one of the main goals of SASS -- don't repeat yourself.

Load the page now, and you should see the links at the bottom in the correct style but on the left of the page. We want them to be inside of the main container, that is to say, we want there to be a margin on the left side so that the links aren't left justified. We will use the bootstrap container class to do this.

    <footer>
      <div class="container">
        <ul>
          <li><a href="#">Help</a></li>
          <li><a href="#">About</a></li>
          <li><a href="#">Sign in</a></li>
        </ul>
      </div>
    </footer>

That's better.

Footer with images

The design has some repeating art on the "top" of the footer, and then a gray background. In order for this to style correctly, we are going to create an additional footer class above the existing one with the list. The first one will contain the artwork, and the second one will contain the links with the gray background.

Add an additional footer for the artwork app/views/layouts/application.html.erb

    <footer class="header"></footer>
    <footer class="body">
      <div class="container">
        <ul>
          <li><a href="#">Help</a></li>
          <li><a href="#">About</a></li>
          <li><a href="#">Sign in</a></li>
        </ul>
      </div>
    </footer>

Append to app/assets/stylesheets/application.css.scss:

$footer-header-height: 102px;
$footer-body-height: 50px;

footer.header {
  bottom: $footer-body-height;
  background: image-url( "footer_art.png" ) repeat-x;
  height: $footer-header-height;
}
footer.body {
  background: image-url( "footer_swatch.png") repeat;
  min-height: $footer-body-height;
}

Here we are using SASS variables to specify the heights of the two footer elements. 102px is the height of the asset image itself, and 50px is just a number that we picked. Using the variables here, we specify that the top element starts at the "top of the bottom element" (i.e. $footer-body-height), that it is $footer-header-height tall, and has a repeating image of the footer_art.png.

The bottom image inherits from the other footer css rule, so it's bottom is still 0, and it is $footer-body-height tall.

Using variables are another way to keep your CSS clean, especially if they are somewhat arbitrary -- you can make sense of what they are used for by choosing a meaningful name. And putting these in variables will come in useful below.

Footer, sticky bottom.

That looks ok, but lets position on the bottom based upon http://getbootstrap.com/examples/sticky-footer/

What we need to do is

  1. Make sure the html document is the "full height" of the window
  2. We have room for the footer in the body, by adding a margin of the height of the footer
  3. Position the footer absolutely, and hard code the height

In our case, since we have 2 footers, we need to position them both, one on top of the other.

Here's the entire app/assets/stylesheets/application.css.scss:

//-----------------------------------------
// Variables, customize bootstrap here
//-----------------------------------------

@import 'bootstrap-sprockets';
@import 'bootstrap';

$footer-header-height: 102px;
$footer-body-height: 50px;

html {
  position: relative;
  min-height: 100%;
}

body {
  background: image-url( "primary_bkg.jpg" ) #0092ce;
  background-repeat: repeat-x;
  margin-bottom: $footer-header-height + $footer-body-height;
}

footer {
  position: absolute;
  bottom: 0;
  width: 100%;
  ul {
    list-style: none;
    padding-left: 0;
    li {
      display: inline;
      margin-right: 1em;
      a {
        text-transform: uppercase;
        font-weight: bold;
        color: white;
      }

      a:hover {
        text-transform: uppercase;
        color: white;
      }
    }
  }
}

footer.header {
  bottom: $footer-body-height;
  background: image-url( "footer_art.png" ) repeat-x;
  height: $footer-header-height;
}
footer.body {
  background: image-url( "footer_swatch.png") repeat;
  min-height: $footer-body-height;
}

Notice that on the body tag, we specify the margin-bottom in terms of the two SASS variables. When the SASS is compiled to CSS, the math will be done by the compiler and will be replaced, in this case, with 152px. If we want to change the bottom to be, say 75px tall, we can change it in one place, and then the values inside of body, footer.header and footer.body will all get updated to consisent values.
