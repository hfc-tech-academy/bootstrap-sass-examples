Home Page Signed In

  • Signed in Header
  • Shoutout Carousel - A carousel of the shoutouts
  • Shoutout navigation - Links to all the user's shoutouts
  • Footer - The bottom page with links

Four things to do here. Lets create another page to do this work in.

Inside of config/routes.rb add a new route

  get '/homepage' => 'landing#homepage'

and lets create an app/views/landing/homepage.html.erb, and we'll drop in the carousel that we've created from the regular page. Notice that we have this in a row on it's own, in a col-md-12 which extends the full width of the container.

<div class="navcircle">
  <ul>
    <li class="home"><a href="#">Home</a></li>
    <li class="friends"><a href="#">Friends</a></li>
    <li class="comments"><a href="#">Comments</a></li>
    <li class="profile"><a href="#">Profile</a></li>
  </ul>
  <img src="http://placehold.it/200x200" >
</div>

<div class="row">
  <div class="col-md-12">
    <%= render partial: "shoutouts" %>
  </div>
</div>

<div class="row">
  <div class="col-md-12">
    <div class="shoutout_nav">
    </div>
  </div>
</div>

Now go to http://localhost:3000/homepage to check it out. It's a start.

That doesn't work. If we go back and look at the layout in application.html.erb, we see that the main <%= yield %> is wrapped in a container div. In this page, both the navcircle and the shoutout navigation are completely left justified. Lets

  1. pull that container div out of the layout
  2. wrap the previous page in a div
  3. explicitly wrap the content with divs when it want it.

So in app/views/layouts/application.html.erb change

    <div class="container">
      <%= yield %>
      <% # Javascript at the bottom for fast page loading %>
      <%= javascript_include_tag 'application' %>
      <% #  Append your own using content_for :javascripts %>
      <%= yield :javascripts %>
    </div>

to

    <%= yield %>
    <% # Javascript at the bottom for fast page loading %>
    <%= javascript_include_tag 'application' %>
    <% #  Append your own using content_for :javascripts %>
    <%= yield :javascripts %>

and wrap the entire page of app/views/landing/index.html.erb in a <div class="container">

Go back to http://localhost:3000 to make sure that there we no regressions.

Edit app/views/landing/homepage.html.erb to put the container element around the carousel.

Now when we go to http://localhost:3000/homepage we see that the nav element, such as it is, is left justified. While the design calls for the navbar to be a circle, we are marking up the links to be a list. We have image assets, and we are going to style each of the list items to have a background image which is explicitly positioned on the page. This will work for larger screens, and will let us adjust the CSS to show something good on smaller screens.

Lets style this up now with the assets:

.navcircle {
  position: absolute;
  ul {
    position: absolute;
    list-style: none;
    margin: 0;
    padding: 0;
    li {
      position: relative;
      a {
        color: transparent;
      }
    }
  }

  .home {
    background-image: image-url( "nav_items/nav_home.png");
    background-repeat: no-repeat;
    height: 74px;
    width: 74px;
    top: 0;
    left: 146px;
    &:hover, &.active {
      background-image: image-url( "nav_items/nav_home_active.png" );
    }
  }
  .friends {
    background-image: image-url( "nav_items/nav_friends.png");
    background-repeat: no-repeat;
    height: 111px;
    width: 82px;
    top: -6px;
    left: 139px;
    &:hover, &.active {
      background-image: image-url( "nav_items/nav_friends_active.png" );
    }
  }
  .comments {
    background-image: image-url( "nav_items/nav_comment.png");
    background-repeat: no-repeat;
    height: 78px;
    width: 104px;
    top: -29px;
    left: 78px;
    &:hover, &.active {
      background-image: image-url( "nav_items/nav_comment_active.png" );
    }

  }
  .profile {
    background-image: image-url( "nav_items/nav_profile.png");
    background-repeat: no-repeat;
    height: 58px;
    width: 84px;
    top: -86px;
    &:hover, &.active {
      background-image: image-url( "nav_items/nav_profile_active.png" );
    }
  }
}

This is what we get:

There's a few things going on. The first is that we get rid of the padding and list style on the ul and li, and make the link text transparent. Then for each of the different buttons, we specify a background image, height, and positioning of where the elements will be shown. We've also set the hover state of the image using the SASS & operator, which is a way to reference the parent element.

.profile {
  &:home, &.active {
   ...
  }
}

Translates to

.profile:home, .profile.active {
  ...
}

Now we just need to put the picture in the space provided. The strategy that we are using, much like positioning the li elements, is to position the element absolutely and tweak the values until it looks correct.

.navcircle {
  ...

  img {
    position: absolute;
    top: -36px;
    left: -53px;
    border-radius: 100%;
    border: 3px solid #0191cd;
    width: 231px;
    height: 231px;
  }
}

Since the .navcircle is position: absolute, its pulled out of the normal flow layout of the page, and now the shoutouts form slides under the nav element. This isn't what we want, so lets add some space into the navspacer to slide it down.

.navspacer {
  padding-top: 250px;
}

That looks better!

A responsive interlude

This nav looks fine on larger screens, but would be a real mess on smaller screens. Lets wrap the entire .navcircle rule inside of a media query that only kicks in when the screen is larger than $screen-sm-min.

@media queries are part of CSS, and they will apply rules based upon the current view point. We are going to set it so all of the CSS we just wrote only applies to screens of a certain width.

@media (min-width: $screen-sm-min) {
  .navcircle {
    position: absolute;

    ul {
      position: relative;
      list-style: none;
      margin: 0;
      padding: 0;
      float: left;
      z-index: 100;
      li {
        position: relative;
        a {
          color: transparent;
        }
      }
    }

    .home {
      background-image: image-url( "nav_items/nav_home.png");
      background-repeat: no-repeat;
      height: 74px;
      width: 74px;
      top: 0;
      left: 146px;
      &:hover, &.active {
        background-image: image-url( "nav_items/nav_home_active.png" );
      }
    }
    .friends {
      background-image: image-url( "nav_items/nav_friends.png");
      background-repeat: no-repeat;
      height: 111px;
      width: 82px;
      top: -6px;
      left: 139px;
      &:hover, &.active {
        background-image: image-url( "nav_items/nav_friends_active.png" );
      }
    }
    .comments {
      background-image: image-url( "nav_items/nav_comment.png");
      background-repeat: no-repeat;
      height: 78px;
      width: 104px;
      top: -29px;
      left: 78px;
      &:hover, &.active {
        background-image: image-url( "nav_items/nav_comment_active.png" );
      }

    }
    .profile {
      background-image: image-url( "nav_items/nav_profile.png");
      background-repeat: no-repeat;
      height: 58px;
      width: 84px;
      top: -86px;
      &:hover, &.active {
        background-image: image-url( "nav_items/nav_profile_active.png" );
      }
    }

    img {
      position: absolute;
      top: -36px;
      left: -53px;
      border-radius: 100%;
      border: 3px solid #0191cd;
      width: 231px;
      height: 231px;
    }
  }

  .navspacer {
    padding-top: 250px;
  }
}

Reload the page, resize the window and see what happens when things get bigger and smaller. When it's large you see our custom navigator, and when it's small all of the styling goes away. Let's add that styling for the smaller screens now:

@media (max-width: $screen-sm-min) {
  .navcircle {
    img {
      display: none;
    }

    ul {
      list-style: none;
      background: #f9f9f9;
      padding: 1em;

      li {
        display: inline;
        margin-left: 1em;
        margin-right: 1em;

        a {
          // font-size: 30px;
          color: #01a84e;
          text-transform: uppercase;

        }

        &.active {
          background: #ccc;
        }
      }
    }
  }
}

Now resize the window. Fancy fancy!

Lets walk through how we are going to style this. We want to have the list be horizontal, so lets first get rid of the list style and make the li's into inline-blocks:

ul.shoutout_nav {
  margin: 0;
  padding: 0;
  list-style: none;

  li {
    display: inline-block;
  }
}

Now lets add the background images for the different style types:

ul.shoutout_nav {
