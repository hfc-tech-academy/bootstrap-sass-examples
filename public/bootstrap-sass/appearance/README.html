Bootstrap-sass on Rails

Appearance

The bootstrap default variables are defined here: https://github.com/twbs/bootstrap-sass/blob/master/assets/stylesheets/bootstrap/_variables.scss

This is a good document to read through to see what can be customized. If you wish to build a static bootstrap.css file for your side using the bootstrap customizer these are the variables that it will let you set.

Let's go through a few key ones to help you see what is all there.

Base Colors

$body-bg is the background of the body tag.

$text-color is the global text color in the body tag, set to $gray-dark

$link-color is set to $brand-primary (described below), and the $link-hover-color is defined at 15% darker (darken($link-color,15%)).

This is a situation where the hover color is dynamically set based upon a previous value, so you can just change the $link-color and it will attempt to pick a good hover color. Or you can set them both manually for finer grained control.

"Brand" Colors

Color Default Example
$brand-primary #428bca
$brand-success #5cb85c
$brand-info #5bc0de
$brand-warning #f0ad4e
$brand-danger #d9534f

Most of the colors that bootstrap uses are derived from these settings. When you change them, the changes will propogate out to all of the buttons, text emphases, labels, alerts, menu and button highlight colors etc.

In general $brand-primary should be the main color of your site, or the dominant color in your design. This sets the link color, hover select states for various components, and the palate for *-primary classes.

prefix What for
btn-* button styles
label-* form labels

These aren't really brand colors, but rather informational colors for the site. Brand colors generally are in the form of a primary color, accent color, and are related to the mood of the design. These are colors to alert the user that there are different types of actions, so their names are a bit misleading.

Where do these show up?

prefix what for
alert-* alerts
bg-* background
text-* text emphasis
panel-* panel heading colors
table-* tables
had-* feedback states

In a rails site, it's common after controller actions to display a flash message saying if that the operation was a success. We can use the alert classes to display this on the top of the layout. Code in the layout could look something like this:

<% flash.each do |key, value| %>
    <div class="<%= flash_class(key) %>" role="alert">
        <button class="close" type=button" data-dismiss="alert">
          <span aria-hidden="true">&times;</span>
          <span class="sr-only">Close</span>
        </button>
        <p><%= value %></p>
    </div>
<% end %>

And a method in application_helper.rb to pull out the correct classes for the flash message.

  def flash_class(level)
    case level.to_sym
        when :notice then "alert alert-info"
        when :success then "alert alert-success"
        when :alert then "alert alert-warning"
        when :error then "alert alert-danger"
        else "alert-danger alert"
    end
  end

In this example if you set a flash[:notice] = "All good here", <div class="alert alert-info"> will get rendered. This uses the alert bootstrap component with the alert-info style. This style is colored with the value from $state-info.

Typography

Bootstrap defines a number of typographical conventions in the official documentation. There's a lot of good stuff here, but the main thing that you want to do is change fonts and sizes.

$font-family-base and $headings-font-family are the main ways to set fonts across the site. By default they are both set to $font-family-sans-serif. Generally you'll want a different font for headings than the main body font.

Font Variables Values
$font-family-base $font-family-sans-serif !default;
$headings-font-family inherit !default;
$font-size-base 14px !default;
-- --
$font-family-sans-serif "Helvetica Neue", Helvetica, Arial, sans-serif !default;
$font-family-serif Georgia, "Times New Roman", Times, serif !default;
$font-family-monospace Menlo, Monaco, Consolas, "Courier New", monospace !default;

Example of setting the $headings-font-family to Open Sans.

@import url(http://fonts.googleapis.com/css?family=Open+Sans:400,300,600);
$headings-font-family: 'Open Sans';

@import 'bootstrap-sprockets';
@import 'bootstrap';

The official documentation has a number of examples for more specific use of types that is worth going through.
