Bootstrap-sass on Rails

Overview

Bootstrap is a framework to build front-ends of websites. Web front ends are composed of three main things: HTML, CSS, and Javascript. Bootstrap is a system that makes it easier to generate these three things.

HTML

HTML stands for HyperText Markup Language, which is roughly the "structure" of your site. It contains all of the content. The content refers to the copy and the ways that the content is structured. A blog post, for example, is primarily a number of paragraphs and texts, but there are also comments, navigational elements, and perhaps a sidebar on the right which are all part of the page. HTML is the language that indicates which part is which.

CSS

CSS stands for Cascading Style Sheets and defines the look of the site. If HTML says there are paragraphs of article text and a section for comments, CSS tells the browser that the article text is on the left side of the screen and the comments are below and in a different font.

Both of these languages are declaritive languages, which means you "declare" to the computer what you want and then let the computer deal with it. You care only about what you want, not the details of how to make it happen. Because of this some people find it jarring to hear the phrase "programming HTML", since programming is alledgely about bossing the computer around rather than merely describing what you want in mind-numbing detail. These people are at best vigoriously punctilious and can safely be ignored. The issue can be avoided by saying that you are "coding HTML".

The other major declaritive langague that you will run into is SQL, which is for getting information out of a database.

JavaScript

The final of the triumverate is JavaScript, a so-called scripting language that lets you add interactivity to the page. Unlike HTML and CSS, which are declaritive languages, JavaScript is imperitive. This means you tell the computer what to do at each step of the way. We will be using JavaScript to control animations, hide and show different elements, alter the structure of the page when a user clicks on something, and to communicate back to the server.

Bootstrap uses jQuery, a JavaScript library which is used by almost 70% of the top sites on the internet. jQuery helps manage the differences between the various browsers so you don't have to. (One of the great things about bootstrap is that it's been tested to work across all the major browsers, so that you can reasonably expect a site developed in Chrome to look the same in Internet Explorer.) The jQuery library is something to get familiar with once you exaust the built-in functionality of bootstrap.

SASS

The version of Bootstrap within the gem are using makes use of SASS and Javascript. SASS makes writing CSS easier by cleaning up the language so that you can express what you'd like more consisely and directly. Writing CSS directly can be a little repetitive and verbose, and SASS is tool that helps clean up this process.

As an aside, the official bootstrap is written in LESS. At the level that we are speaking of it, LESS is the same as SASS. They are two seperate projects that share the same goal — a better way of writing CSS — and the differences between them are largely a matter of style.

Since its job is to make writing CSS easier, SASS looks a lot like CSS. In fact all valid CSS is valid SASS.

Putting it all together

Bootstrap, altogether, is a bunch of SASS and JavaScript files that you include in your site.

You write HTML and reference classes and components that bootstrap defines for you.

You add custom SASS and CSS to change the look — and perhaps extend the inner workings — of the base framework.

By adding HTML data elements to tags, you plug into the JavaScript framework provided by bootstrap. You also can add your own JavaScript to further extend what the site does.
