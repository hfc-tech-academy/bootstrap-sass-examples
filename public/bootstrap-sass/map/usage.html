Bootstrap-sass on Rails

Usage

There are a number of different ways to use the bootstrap framework.

Using the precompiled CSS

The simpliest way is to simply drop the bootstrap.min.css, bootstrap.theme.css, and bootstrap.min.js into your project and put your customizations into additional CSS. You can't customize the framework itself when you use it this way, but you have a base environment with which to start building your site. You get all the typography and normalization clean up and can use all of the components and features, but you need to use the stock settings for everything.

There's a simplicity to this. You get responsive layout using the grid layout and you are also working in a richer styling environment than when using CSS directly.

This simplicity means you need to work within bootstrap's defaults, which are perfectly fine but may not fit your design. You'll probably end up modifying the design to use what bootstrap provides by default, from whence comes the complaint that "all bootstrap sites look the same" — this is because most people just use the defaults!

Customizing into static files

A slightly more flexible method is to use a customized version of bootstrap. You pick and choose which elements you want to include and change the parameters of the components to match your design more closely.

There's a section in the official documentation that lets you recompile and download a custom version of bootstrap: http://getbootstrap.com/customize/

Once you select the components that you want, the site generates a version of these files for you, which you then include into your build.

Customizing using SASS/LESS

The most complete way of using bootstrap — which this document describes — is to use the SASS/LESS files directly. This is very similar to using the official customization tool, but it also gives you access to all of the mixins and lets you define semantic classes: classes that have the same functionality as the bootstrap classes but are named something meaningful to your application.

Since we are internally using the same method to generate the customizations as the offical web tool, the documentation applies to both. The sequence of steps for using customized bootstrap in your project is:

  1. Specify your customizations to bootstrap variables
  2. Include the bootstrap SASS (it automatically uses the defaults for any variables you haven't specified)
  3. Add your custom styles
  4. Apply to HTML

Variables control any modifications to bootstrap. For example, if your business model is centered around the color aquamarine, then setting the $brand-primary variable to #7FFFD4 in Step 1 will tell bootstrap when it's included in Step 2 to use that color for buttons, backgrounds, accents, and any other place it thinks your brand color should show up. If your brand color is aquamarine but you really want your buttons to be more of a teal color, then you can override that in Step 3 with specific rules that change the look in ways that bootstrap's variables don't necessarily allow.
