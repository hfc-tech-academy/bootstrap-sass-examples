Advanced Techniques

Lets look at a design for a site where you have navigation on the left, and a content area on the right:

This is designed on the grid, with 2 columns for a nav, 2 more columns for the icons and avatars, and the remaining 8 for the verbage.

How would you style this?

The vertical rythms of the left side of the page and the right side of the page don't match up, and while we know that the right side will be taller than the left, we don't really know where. How do we add the other rows?

From this we can see that using the bootstrap classes at this point will only approximate the design. We could do three things:

  • call 2-10 close enough
  • call 3-9 close enough
  • Figure out how to redefine the grid to do what we want

To make it look correct, what we need to do is to define the grid inside of that 10-width column to be 10 columns wide.

Let's write the html to use our own classes, which we can then define.

<div class="container">
  <div class="row"> <!-- This is what we want it to look like -->
    <div class="col-md-2 bordered">col-md-2</div>
    <div class="col-md-2 bordered">col-md-2</div>
    <div class="col-md-8 bordered">col-md-8</div>
  </div>

  <div class="content">
    <div class="sidebar">
      <ul>
        <li>Stuff</li>
        <li>Stuff</li>
      </ul>
    </div>
    <div class="main">
      <div class="row">
         <div class="left">Icon</div>

         <div class="left">Icon</div>

Now lets get busy with the SCSS

.content {
  @extend .row;
  @extend .bordered;

  .sidebar {
    @extend .col-md-2;
    @extend .bordered;
  }

  .main {
    @extend .col-md-10;
    @extend .bordered;

    $grid-columns: 10;
    .left {
      @include make-sm-column( 2 );
      @extend .bordered;
    }

    .right {
      @include make-sm-column( 8 );
      @extend .bordered;
    }
  }
}

@include runs a SASS function to generate the properties that get included. This lets you pass in parameters of different values. The make-md-column function is part of the bootstrap grid system and lets us generate the properties needed for a 2 column md grid. (The functions are defined here: https://github.com/twbs/bootstrap-sass/tree/master/assets/stylesheets/bootstrap/mixins/_grid.scss )

We're mixing up a few different things here to get our result, so picking it apart is actually quite tricky. Lets go through what's happening:

  1. When you compile the bootstrap sass, it calls the $make-grid function.
  2. This in turn generates all of the col-sm-*, col-md-*, etc class definations based upon $grid-columns.
  3. The methods that it uses are named make-sm-column, make-md-column, etc. These define the column properties in terms of percentage of over all width.
  4. When our code is run, these classes are available (col-md-2 and col-md-10 in our case) to extend.
  5. We then call the functions make-sm-column with the parameter 2 to genereate the properties for a 2 columns.
  6. Instead of referencing these already generated properties, we first change the number of columns and generate a new set of properties for our classes.
